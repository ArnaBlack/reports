anchor-name: --my-anchor;

Умные fallbacks:

position-try-*

			
				.target {
					position: fixed;
					position-try-fallbacks: --my-custom-position,
					--my-custom-position flip-block,
					bottom left;
				}
			
		
			
				.target {
					/*try-tactic*/
					position-try-fallbacks: flip-block;
				}
			
		
			
				.target {
					/*position-area*/
					position-try-fallbacks: top right, bottom left;
				}

Поведение

  • position: fixed — фолбэк срабатывает у края экрана
  • position: absolute — фолбэк срабатывает у края родителя
  • Для absolute родителю нужно указать overflow, иначе фолбэки (position-try) не запустятся 
			
				.target {
					position-area: top;
					position-try-fallbacks: flip-block,
											flip-inline, 
											flip-block flip-inline;
				}
position-try-order:
normal |
most-width |
most-height |
most-block-size |
most-inline-size

Проблема:
тултип умный, а стрелка — нет

Решение

@container anchored(fallback) 

			
				.target {
					/* тут стили для позиционирования */
					/* ... */
  					container-type: anchored;

					&::before {
						content: '▲';
						position: absolute;
						bottom: 100%;
					}
				}
			
		
			
				@container anchored(fallback: flip-block) {
					.target::before {
						content: '▼';
						bottom: auto;
						top: 100%;
					}
				}

    Итоги и выводы Anchor Position API

    • Что это?
      Мощнейший API для позиционирования на чистом CSS
    • Что решает?
      Избавляет от тонн хрупкого JS-кода для поповеров, тултипов, меню
    • В чем преимущество?
      Производительность, надежность, декларативность
    • Когда использовать?
      Уже сейчас для прогрессивного улучшения.
      В будущем — как стандартный подход
